# Условия:

+ Используй React (версия 18+).
    `react + nextjs для прокси запросов к https://dummyjson.com/ (у меня только под vpn открылся)`
    `тоже самое и для загрузки картинок - использовал rewrites в настройках nextjs`
+ Используй TypeScript (строгая типизация обязательна).
+ Работоспособность в актуальной версии Google Chrome.
+ Остальное (выбор стейт-менеджера, UI библиотек) на твоё усмотрение, но будь готов обосновать выбор.
    `mobx классика, useQuery, tailwindcss для скорости прототипирования`

# Дизайн:

+ Реализация должна визуально соответствовать макету в Figma. 
    `макет отличается в ворде и фигме, макет кривоват в фигме - логику css переменных не проследить, все кривоватеноко`
    `поэтому делал не pixel-perfect, и цвета кое где вместо рандомных из фигмы - похожие`

# Функциональные требования:

+ Форма входа:
    `не по макету добавил компоненту User в угол, для удобных тестов`
    + Валидация полей (обязательность заполнения).
    `валидацию минимальную поставил, т.к. требований нет. можно было бы какие-нибудь библиотеки заюзать типо zod`
    + Обработка ошибок: если API возвращает ошибку, выводить уведомление или текст ошибки под полями.

+ Логика запоминания данных для входа:
    + Если чекбокс установлен, нужно сохранять токен авторизации так, чтобы сессия жила после закрытия браузера
    `использовал localStorage`
    + Если не установлен, то сессия должна сбрасываться при закрытии вкладки
    `использовал sessionStorage`

+ Вывод списка товаров:
    + Соответствие столбцам из макета Figma
    `сделал конфигурируемый конфиг таблицы products.config.ts и темплэйты для отображения нестандартных cell`

    + Прогресс-бар при подгрузке
    `fontawesome spinner и пустые строки чтобы не прыгала таблица на загрузке`

    + Подгрузка данных из API.
    `иногда запрос к dummyjson валится, если быстро кликать, использовал useQuery для повторных попыток и незаметности для юзера`
    `возвращаю только title,brand,sku,rating,price,category,thumbnail`

+ Сортировка:
    Возможность сортировки по столбцам (например, по цене или рейтингу)
    `сделал сортировку по всем колонкам с данными`

    Должно храниться состояние сортировки 
    `использовал localStorage (можно как альтернативу было взять mobx-persist-store)`

- Добавление товара:
    По нажатию кнопки "Добавить" открывается форма добавления товара с возможностью заполнить основные поля: Наименование, цена, вендор, артикул.
    При успешном добавлении  показывать базовое Toast уведомление 
    Логику сохранения через API делать при этом не нужно.

+ Логика интерфейса:
    Если рейтинг товара ниже 3, значение должно подсвечиваться красным цветом.

+ Поиск товаров:
    Использовать API
    `useDebounce`
